# 舆情监控与报告生成

## 概述

针对艺人、影视作品、品牌等对象，在指定时间范围内采集公开舆情信息，
生成结构化的舆情监测报告，输出为可预览的 HTML 网页文件。

## 触发条件

当用户请求涉及以下关键词时触发此 Skill：
- "舆情"、"口碑"、"舆论"、"风评"
- "舆情监测"、"舆情报告"、"舆情跑一下"
- "XX最近口碑怎么样"、"XX舆论分析"

## 监测对象

支持以下类型：
- **艺人/演员**：姓名 + 相关作品关键词
- **影视作品**：剧名/片名 + 播出平台
- **品牌/产品**：品牌名 + 行业关键词

## 工作流程

### Step 1: 确认参数

从用户请求中提取以下参数，缺失项使用默认值：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 监测对象 | 用户指定 | 艺人名/品牌名 |
| 时间范围 | 最近30天 | 用户指定则按指定范围 |
| 输出格式 | HTML网页预览 | 用户要求PDF则额外生成 |
| 关注维度 | 全维度 | 粉丝数据、口碑事件、平台热度、风险预警等 |

### Step 2: 多源信息采集

使用 WebSearch 和 WebFetch 工具，分以下维度进行信息采集：

**2.1 基础数据采集**（至少执行3-5次搜索）
- 搜索词示例：
  - `"{对象名} {起始日期}-{结束日期} 新闻"`
  - `"{对象名} 最新动态 热搜"`
  - `"{对象名} 口碑 评价 争议"`
  - `"{对象名} 粉丝数据 涨粉"`
  - `"{对象名} 微博 抖音 小红书 热度"`

**2.2 热点事件采集**
- 搜索近期的热点话题、热搜事件
- 识别正面事件（作品播出、获奖、公益等）
- 识别负面事件（争议、吐槽、翻车等）
- 记录事件时间、涉及平台、舆论走向

**2.3 平台数据采集**（如可获取）
- 微博：热搜次数、话题阅读量、互动量
- 抖音：相关视频播放量、点赞量
- 小红书：相关笔记数、互动量
- 其他：百度指数、微信指数等

### Step 3: 数据分析与研判

**3.1 情感分析**
根据采集到的信息，判断整体舆情情感分布：
- 正面舆情比例（好评、支持、喜爱）
- 中性舆情比例（客观报道、讨论）
- 负面舆情比例（批评、争议、抵制）

**3.2 热点事件梳理**
按时间线整理主要舆情事件（不少于5个），每个事件包含：
- 事件名称
- 发生时间
- 舆论倾向（正面/中性/负面）
- 影响程度（高/中/低）
- 简要描述

**3.3 风险评估**
识别当前存在或潜在的风险点：
- 正在发酵的负面舆情
- 可能引发争议的敏感话题
- 竞品对比中的劣势

### Step 4: 生成 HTML 报告

使用 `references/report_template.html` 作为样式参考，
生成完整的 HTML 舆情报告文件。报告文件命名规则：
`{对象名}{天数}天舆情报告_{YYYYMMDD}.html`

**报告结构**（必须包含以下所有模块）：

```
1. 报告头部
   - 报告标题：{对象名} {天数}天舆情监测报告
   - 监测周期：{起始日期} - {结束日期}
   - 生成日期：当天日期

2. 核心数据概览（KPI卡片）
   - 整体舆情评分（0-100）
   - 正面/中性/负面舆情占比
   - 热搜/热点事件数量
   - 主要活跃平台数

3. 舆情情感分布
   - 饼图或进度条展示正面/中性/负面比例
   - 情感趋势描述

4. 重大舆情事件时间线
   - 按时间排列的事件列表
   - 每个事件标注舆论倾向和影响程度

5. 平台表现分析
   - 各平台热度对比
   - 主要讨论话题分布

6. 风险评估矩阵
   - 当前风险点列表
   - 风险等级（高/中/低）
   - 应对建议

7. 运营建议
   - 短期（1-2周）：即时应对策略
   - 中期（1个月）：口碑维护计划
   - 长期（3个月+）：形象提升方案

8. 免责声明
   - 数据来源说明（公开网络信息）
   - 报告仅供参考
```

**HTML 样式要求**：
- 使用内联 CSS，不依赖外部样式表
- 响应式设计，支持手机端查看
- 配色方案：主色 #1a1a2e，辅色 #16213e，强调色 #0f3460，正向 #27ae60，负向 #e74c3c，中性 #f39c12
- 中文字体栈：-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif
- KPI 数据使用卡片式布局
- 事件时间线使用竖向时间轴样式

### Step 5: 交付与呈现

1. 使用 `preview_url` 工具打开 HTML 报告预览
2. 使用 `deliver_attachments` 工具将 HTML 文件发送给用户
3. 如用户要求 PDF，额外使用 reportlab 生成 PDF 并发送

## 注意事项

- 所有数据必须来自公开网络信息，不得编造数据
- 如果某个维度的数据无法获取，在报告中明确标注"数据暂缺"
- 时间范围必须根据当天日期自动计算，确保准确性
- 情感分析基于采集到的信息进行合理研判，避免主观臆断
- 报告中避免涉及政治敏感内容
- 对于争议性事件，保持客观中立，呈现多方观点
